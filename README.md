# Customer Churn ETL Pipeline

## 📌 Project Overview

This project implements a **serverless, cost-effective ETL pipeline** for Customer Churn analysis using AWS-native services and visualization in Power BI. It orchestrates ingestion, cataloging, transformation, querying, and dashboarding of customer data.

---

## 🔧 Technologies Used

- **Amazon S3**: Raw and processed data storage
- **AWS Glue**: Data catalog and ETL transformation
- **Amazon Athena**: Serverless querying of processed data
- **Power BI**: Business intelligence and dashboards
- **Apache Airflow**: Pipeline orchestration
- **Python (Airflow DAGs)**

---

## 🗂️ Pipeline Architecture

1. **Data Ingestion**: Raw files stored in an S3 bucket
2. **Data Cataloging**: AWS Glue crawler populates the Data Catalog
3. **Data Transformation**: Glue ETL job processes data and stores it as Parquet
4. **Data Querying**: Athena reads optimized data from S3
5. **Visualization**: Power BI consumes Athena queries via ODBC
6. **Automation**: Apache Airflow automates the process through DAGs

---

## 🚀 Setup Instructions

### Step 1: AWS Environment Preparation

- Create 3 S3 buckets:
  - `raw-data-bucket`
  - `processed-data-bucket`
  - `athena-query-results-bucket`
- Create IAM Roles:
  - **Glue Role** with `AmazonS3FullAccess`, `AWSGlueServiceRole`, `AmazonAthenaFullAccess`
  - **Power BI IAM User** with Athena access and permission to read the results bucket

---

### Step 2: AWS Glue Configuration

1. **Create a Glue Database** (e.g., `my_etl_database`)
2. **Add a Crawler** pointing to the raw S3 data
3. **Run the Crawler** to populate your Data Catalog

---

### Step 3: ETL Job in Glue

- Create and configure a Spark job (`transform_raw_data`)
- Use the table generated by the crawler as input
- Transform and output to `processed-data-bucket` in **Parquet** format
- Monitor execution and logs via Glue Console

---

### Step 4: Athena Setup

- Configure Athena to use your results S3 bucket
- Use SQL DDL to create external tables pointing to the processed data

---

### Step 5: Power BI Integration

1. **Install the Amazon Athena ODBC Driver**
2. **Configure a System DSN** with your region, S3 output path, and IAM credentials
3. **Connect Power BI Desktop**:
   - Go to *Get Data > ODBC*
   - Choose your Athena DSN
   - Select tables and choose Import or DirectQuery mode

[📘 Athena & Power BI Docs](https://docs.aws.amazon.com/athena/latest/ug/connect-with-odbc-and-power-bi.html)

---

### Step 6: Orchestration with Airflow

- **Use Apache Airflow (or Amazon MWAA)**
- Implement DAG in:
  ```bash
  /home/ubuntu/airflow/dags/glue_cust_churn_etl.py
  ```
- DAG Responsibilities:
  - Detect new files in raw S3
  - Trigger Glue job when new data arrives
